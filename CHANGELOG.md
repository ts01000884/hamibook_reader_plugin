# 更新紀錄

版本規則：`1.x` 為 **Chrome 擴充版**；`0.x` 為改寫前 **Tampermonkey 使用者腳本時期**的更新軌跡（原標為 `v1.x`，改寫成擴充後重新編為 `0.x` 保留），日期以合併整理日 2026-07-06 為準。

---

## Chrome 擴充版

### 1.0.0 — 2026-07-06 · 首發
由 Tampermonkey 使用者腳本改寫為 Chrome 擴充（Manifest V3），功能與 `0.7.0` 相同，僅載體不同：

- 生效網址 / 執行時機 / 只在頂層執行，改由 `manifest.json` 宣告（對應原 `@match` / `@run-at document-start` / `@noframes`）。
- 以 `world: "MAIN"` 的 content script 執行，維持與原腳本 `@grant none` 相同的頁面環境行為。
- 移除 `GM_registerMenuCommand` 依賴（Tampermonkey 專屬 API）；黑夜模式改由本來就有的右下角浮動按鈕切換。
- 清除 `// ==UserScript==` 標頭，改為擴充用註解。
- 新增擴充圖示與審查說明（README）。
- **不要求任何權限、無網路連線、無資料收集。**

## Tampermonkey 使用者腳本時期（改寫前，原 v1.x）

### 0.7.0 — 多分頁朗讀協調
修正多分頁同時朗讀會混在一起：Chrome 的 `speechSynthesis` 佇列為整個瀏覽器共用，非各分頁獨立。新增 `BroadcastChannel`，任一分頁開始/恢復朗讀時廣播，其他正在朗讀的分頁收到後自動停止，確保同一時間只有一個分頁出聲。

### 0.6.1 — 收合鈕樣式對齊
收合後的「讀」圓形按鈕，樣式比照黑夜模式切換鈕（相同邊框、陰影、字級/字重），視覺上與「日/夜」按鈕等大。

### 0.6.0 — 面板收合與位置調整
(a) 展開時面板整體上移到 `bottom: 76px`，讓縮放把手遠離按鈕、不搶滑鼠事件。
(b) 新增面板收合/展開鈕（標題列的「－」）；收合後變成與黑夜模式鈕同大的圓鈕，固定於其左側（`right: 72px`），不重疊。收合狀態會記住。

### 0.5.0 — 面板 UI 調整
(a) 面板加上 `resize: both` + `overflow: auto`，右下角可拖曳縮放。
(b)「段落」「已存位置」改用原生 `<details>/<summary>`，預設收合、點擊展開，平常畫面更乾淨。

### 0.4.0 — 架構重寫：念什麼／要不要翻頁分離
`currentChunks` 一律為「整章」段落清單，只在真正換章節時重抓一次；章節中途不重抓、不歸零 `currentIndex`。每個 chunk 保留原始 DOM 元素，念下一段前檢查其是否可見，看不到才點下一頁同步畫面（不影響念到第幾段）。整章念完才觸發換章。移除「翻頁後沒文字就自動跳過」邏輯，純圖片頁不再誤判成「沒內容 / 從頭重念」。

### 0.3.0 — 自動翻頁兩項修正
(a) 新增 `ownTriggeredTurn` 旗標：自己觸發的翻頁交由自動翻頁流程接手續讀，不被舊的「iframe load 就 stopReading」手動翻頁監聽打斷；使用者手動翻頁維持原行為。
(b) 翻到純圖片無文字頁時，立即再點下一頁跳過去找文字（最多連跳 8 頁），不再空等 6 秒。

### 0.2.1 — 下一頁選擇器修正
實際結構為 `<div class="next-block"><a title="下一頁" class="btn next"></a></div>`，改以 `.next-block .btn.next` 優先比對，原猜測的 `.btn-block-next` 保留為備援。

### 0.2.0 — 黑夜模式切章全黑修正
切章時內文 iframe 會整份換新 document，舊注入的樣式/class 隨之消失，導致黑底疊上未上色的原生黑字。改用 `MutationObserver` 持續監看新 iframe，並在其 `load` 事件重新套用樣式。

### 0.1.0 — 自動翻頁接續朗讀
可見範圍或整章讀完時，自動點擊下一頁按鈕，確認內容變更後重抓段落並接續播放。找不到按鈕或翻頁逾時（6 秒）則停止並顯示狀態。可用面板核取方塊關閉。

### 0.0.1 — 初始合併版
合併原本兩支腳本（TTS 朗讀 + UI 黑夜模式）：統一使用較廣的 `@match`，並在 TTS 模組加上 `isTtsSupportedPage()`，維持「TTS 面板只在 EPUB 文字格式 viewer/08 顯示」。統一 `@run-at document-start`，TTS 的 `init()` 加上「等 body 準備好才執行」保護。語速上限鎖定 1.5。
